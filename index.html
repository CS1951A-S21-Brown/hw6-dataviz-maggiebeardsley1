Football Analysis


Note that a big win is defined as a game won by more than 2 goals.

My first graph shows the number of football games by year. For each of the five years I show, I have a bar displaying the number of football games corresponding to that year. My second graph displays winning percentage for the top 10 nations. The size of the circle corresponds to the winning percentage, where a bigger circle represents a larger percentage of games won. Additionally, the colors are darker for higher winning percentage and lighter for lower winning percentage. By hovering over a circle you can see the exact winning percentage. The third graph shows which teams have been highest performing in the 2014 and 2018 world cups by showing winning percentage as bars. The viewer can choose to view "big win percentage" as well, which is the percentage of games won by at least 2 goals. This can be useful for understanding the strength of the team.

D3 was helpful and improved my visualization because there are a lot of options for types of graphs that can be built, and there is a lot of flexibility for how exactly I choose to build those types of graphs. A third reason D3 was helpful is because there are a lot of examples on the D3 website that can be used for inspiration.

D3 was not the best tool because it seems like it is way more difficult than it should be. Especially for a simple bar graph! Additionally, it is time consuming to build graphs in D3, I don't think my dashboard really should have needed > 300 lines of code to build. A third reason why D3 might not be the best tool is it seems like one of the best parts of D3 is some of the really crazy and complicated visualizations that can be built, but oftentimes these are not the best way to portray the information to an audience.

Something I think I did well in terms of accessibility is many of my graphs still portray the necessary information to colorblind people. For example, the bar graphs embed the same information as the color in the length of the bar, and the circle color information is portrayed in the size of the circle. So colorblind people would find this dashboard accessible. Someone who might have more difficulty viewing my dashboard would be blind people. The Schepers reading talked about the importance of making graphs compatible with screen readers, but I don't think I have a good enough understanding of how screen readers work to configure my alt text properly.

I could have included a link to the raw data in my dashboard however since its on the assignment page anyone looking at my assignment could view it there. Others factors that kept me from taking additional accessibility steps are a lack of understanding of the necessary accommodations and a lack of time.
